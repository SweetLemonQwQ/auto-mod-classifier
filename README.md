# 自动筛选模组分类器 2.00

一个面向 Windows 的 Minecraft 模组筛选和自动建服工具。

`2.00` 是当前 GitHub 的对外发布版本号，用来和早期 `1.0` 半成品区分。这个仓库里的源码已经整理到目前这套稳定实现，对外介绍也统一按 `2.00` 发布。

## 这次 2.00 对外版的重点

- 已支持 `MC百科` 验证码页面版本
- 联网查询会改走真实浏览器，遇到验证码时可手动完成后继续
- `CurseForge` 已接入兜底查询，但当前仍按 `测试版` 对待
- 支持 `2次筛选`，只重试首轮 `unknown` 模组
- 在保留模组筛选能力的同时，已经整合 `一键制作服务端模式`

## 主要功能

- `Mod筛选模式`
  - 自动分类为 `服务端保留 / 纯客户端 / 待人工确认`
  - 支持 `Fabric / Quilt / Forge / NeoForge`
  - 支持本地元数据判断、`Modrinth` 联网判断、`MC百科` 兜底判断
  - 可选 `CurseForge` 兜底查询，当前为测试版
  - 输出 `JSON / CSV / TXT` 报告，并单独标记损坏 Jar
- `一键制作服务端模式`
  - 选择整个客户端实例目录和新的空输出目录
  - 自动识别 `Minecraft 版本 + 加载器类型 + 精确加载器版本`
  - 支持 `Fabric / Forge / NeoForge` 自动建服
  - 自动下载官方安装器、复制可用模组、复制常见配置目录
  - 自动生成启动脚本，自动首启，写入 `eula=true`，并做二次启动验证

## 联网筛选说明

- `MC百科`
  - 现在支持带验证码的页面
  - 程序会调起真实浏览器处理拦截或验证码
  - 适合当 `Modrinth` 无法明确判断时作为兜底来源
- `CurseForge`
  - 当前入口已经提供，但速度较慢
  - 部分环境需要梯子
  - 页面结构和拦截机制仍可能变化，所以暂时标记为 `测试版`

## 仓库内容

- `自动筛选模组分类器.pyw`
  - 主程序源码
- `启动自动筛选模组分类器.bat`
  - 源码版启动脚本
- `打包发布.bat`
  - 本地重新打包，并额外生成 GitHub release 用的 exe 文件名
- `自动筛选模组分类器.ico`
  - 程序图标

## 直接运行源码

需要本机已安装 Python 3。

```powershell
pythonw .\自动筛选模组分类器.pyw
```

或者直接双击：

```text
启动自动筛选模组分类器.bat
```

## 重新打包

直接运行：

```text
打包发布.bat
```

脚本会调用 `PyInstaller` 生成单文件 exe，并在 `dist` 目录下同时产出：

- `自动筛选模组分类器.exe`
- `auto-mod-classifier-2.00.exe`

其中第二个文件名可直接用于 GitHub Release 附件上传。

## 注意事项

- 模组联网分类依赖网络，速度会受 `Modrinth / MC百科 / CurseForge` 请求情况影响
- 一键建服不会修改客户端源目录
- 如果版本清单中无法可靠识别出精确加载器版本，程序会直接报错，不做模糊推断
- `Quilt` 当前只保留模组筛选识别，不进入自动建服流程
